# Welcome to the handbook

This is the company's internal knowledge base. Everything here is a plain
markdown file in Git — when something changes, edit the file, open a merge
request, and the handbook updates on deploy.

## How it's organized

- **Getting Started** — your first week, accounts, and tools.
- **IT** — hardware, software, and access how-tos.
- **Policies** — the rules we've written down so nobody has to guess.

## Contributing

Found something outdated? You fix it. The handbook lives in the repository
under `content/kb/` — every article is a `.md` file with a small frontmatter
block (title, category, order). Your edit goes through review like any other
change.
